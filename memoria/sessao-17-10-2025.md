## 🔧 Correções Implementadas

### 1. Problema do useMemo (users/page.tsx)
**Issue:** Ao fazer logout, mostrava apenas último usuário em vez de todos.

**Causa:** useMemo lia localStorage dentro dele, estados não resetavam.

**Solução:**
```typescript
// ANTES
const filteredUsers = useMemo(() => {
  const session = localStorage.getItem('2fa_session')
  // ... lógica complexa
}, [users, currentUserId, currentUserRole])

// DEPOIS
useEffect(() => {
  fetchUsers() // Sempre busca todos
}, [])
// Removido useMemo completamente
```

**Resultado:** ✅ Sempre mostra todos os 3 usuários

---

### 2. Filtro de Arquivos por s3_prefix

**Issue:** Todos viam todos os arquivos, sem controle de acesso.

**Implementação:**
1. Adicionado `s3_prefix` no DynamoDB
2. Incluído `s3_prefix` e `role` no JWT payload
3. Lambda files-handler filtra por prefixo
4. Frontend envia token JWT no header

**Arquivos modificados:**
- `lib/aws-client.ts` - Header Authorization
- `auth-handler/lambda_function.py` - JWT com s3_prefix e role
- `files-handler/lambda_function.py` - Filtro S3 por prefixo
- `update-users-s3-prefix.py` - Script para atualizar DynamoDB

**Mapeamento:**
```
user_admin → s3_prefix="" (vê tudo)
Lid → s3_prefix="Lid/" (vê apenas Lid/*)
Sergio_sena → s3_prefix="Sergio Sena/" (vê apenas Sergio Sena/*)
```

**Status:** ⚠️ Implementado, aguardando teste do usuário

---

### 3. Senha Hardcoded Removida

**Issue:** Admin tinha senha hardcoded na Lambda (má prática).

**Solução:**
1. Script `add-admin-password.py` - Hash SHA256 de "admin123"
2. Atualizado DynamoDB com password hash
3. Removido bloco hardcoded da Lambda auth-handler
4. Única fonte de verdade: DynamoDB

**Antes:**
```python
ADMIN_USER = {"email": "sergiosenaadmin@sstech", "password": "sergiosena"}
if email == ADMIN_USER['email'] and password == ADMIN_USER['password']:
    # ...
```

**Depois:**
```python
# Apenas scan DynamoDB
response = table.scan()
for user in users:
    if user.get('email') == email and user.get('password') == hash_password(password):
        # ...
```

**Resultado:** ✅ Senha segura no DynamoDB

---

### 4. Sessão 2FA Aumentada

**Issue:** Sessão expirava em 5 minutos, pedia re-autenticação frequente.

**Solução:**
```typescript
// ANTES
if (!session || (Date.now() - parseInt(session)) > 300000) { // 5min

// DEPOIS
if (!session || (Date.now() - parseInt(session)) > 1800000) { // 30min
```

**Arquivos:** dashboard/page.tsx, admin/page.tsx

**Resultado:** ✅ Experiência melhor, menos interrupções

### 6. Limite de Upload Aumentado

**Issue:** Usuário precisa enviar arquivo de 7,08GB.

**Solução:**

**Arquivos:** DirectUpload.tsx, dashboard/page.tsx

**Observação:** Usuário questionou se multipart seria melhor. Resposta: Sim, para >5GB é mais confiável. Planejado para v4.5.

## 💬 Decisões Técnicas

### Por que remover useMemo?
- Complexidade desnecessária para caso de uso simples
- Sempre mostrar todos os usuários é o comportamento esperado
- Menos bugs, código mais simples

### Por que 30 minutos de sessão?
- Balanço entre segurança e UX
- Usuário não precisa re-autenticar a cada 5 minutos
- Ainda expira para segurança

### Por que não implementar multipart agora?
- Usuário pediu para documentar primeiro
- Criar ponto de restauração antes de mudanças grandes
- Multipart é feature isolada, pode ser adicionada depois
