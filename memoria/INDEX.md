## 🔄 Fluxo de Trabalho

### Durante Desenvolvimento
1. Criar arquivo em `ATUAL/` com nome descritivo
2. Documentar implementações em tempo real
3. Atualizar este índice

### Ao Finalizar Sprint/Sessão
1. Mover arquivo de `ATUAL/` para `HISTORICO/`
2. Criar subpasta com data se necessário
3. Atualizar CHANGELOG.md
4. Atualizar este índice

### Documentos de Referência
- Manter na raiz
- Atualizar conforme necessário
- Não mover para HISTORICO
